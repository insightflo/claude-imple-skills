# SI Planning — 도메인 프로파일

> **목적**: 도메인별 필수 NFR, 규정 체크리스트, 흔한 누락 요구사항을 사전 로드하여
> Gap Detection의 정확도를 높인다.
>
> **사용**: Phase 0에서 도메인 선택 시 해당 프로파일을 자동 로드.
> `커스텀` 도메인 선택 시 사용자와 함께 체크리스트를 직접 작성.

---

## 도메인 1: 공공 (Public Sector)

### 개요

```
대상: 중앙부처, 지자체, 공공기관, 교육기관, 국방(민수)
핵심 특성: 규정 준수 최우선, 감사 추적 필수, 전자정부 표준 준수
주요 법규: 전자정부법, 개인정보보호법, 정보보안관리규정,
           행정안전부 정보시스템 구축·운영 지침
```

### 필수 NFR

```
[PUB-NFR-01] 웹 접근성
  - 기준: WCAG 2.1 AA 이상 (국내: 웹접근성 지침 2.1)
  - 검증: 자동화 도구(axe) + 전문가 수동 검사
  - 대상: 모든 공개 화면
  - 측정: 접근성 위반 0건 (Critical), 5건 이하 (Serious)

[PUB-NFR-02] 전자정부프레임워크 호환성
  - 기준: eGovFramework 4.x 이상 권장
  - 적용: 백엔드 레이어 (UI 프레임워크는 별도)
  - 예외: 발주처 자체 프레임워크 정책 우선

[PUB-NFR-03] 개인정보보호
  - 법규: 개인정보보호법, 동법 시행령
  - 요건: 개인정보 수집·이용 동의, 처리방침 게시,
          개인정보 처리일지 유지, 파기 정책
  - ISMS-P 인증 대상 여부 확인 필수

[PUB-NFR-04] 정보보안
  - 최소 요건: SSL/TLS 1.2 이상, 패스워드 복잡도 정책,
              세션 타임아웃, 취약점 점검 (연 2회)
  - 공공 클라우드: 클라우드 보안인증(CSAP) 적용 여부

[PUB-NFR-05] 가용성
  - 기본: 연중무휴 (365×24), 계획 점검 시간 제외
  - 서비스 등급에 따라: 상(99.9%), 중(99.5%), 하(99%)
  - RTO: 4시간 이내 (서비스 등급 기준)
  - RPO: 24시간 이내 (서비스 등급 기준)

[PUB-NFR-06] 감사 추적 (Audit Trail)
  - 모든 데이터 변경에 대한 이력 보존
  - 보존 기간: 최소 3년 (법정 기록물 5년)
  - 접근 로그: 사용자, 일시, 작업, 대상 데이터, IP
  - 감사 로그 위변조 방지 (별도 저장 또는 전자서명)

[PUB-NFR-07] 전자서명 / 공인인증
  - GPKI (정부 PKI) 연동 여부 확인
  - 행정전자서명 (GSIGN) 적용 범위
  - 민간 전자서명 인정 여부 (금융결제원 등)
```

### 필수 체크리스트

```
[PUB-CK-01] ✅ 전자결재 / 결재선 기능 정의
             - 전결, 대결, 협조, 수신 결재 유형
             - 결재선 저장/불러오기
             - 결재 반려 및 재상신 흐름

[PUB-CK-02] ✅ 권한 위임 기능
             - 출장/병가 시 권한 대리인 지정
             - 위임 기간 설정 및 자동 만료
             - 위임 이력 기록

[PUB-CK-03] ✅ 메뉴/화면 접근 권한 관리
             - 직급/직책/부서 기반 권한
             - 역할(Role) 기반 접근 제어 (RBAC)
             - 권한 변경 이력

[PUB-CK-04] ✅ 감사 로그 (전 항목 필수)
             - 로그인/로그아웃 이력
             - 데이터 조회/입력/수정/삭제 이력
             - 개인정보 다운로드 이력
             - 권한 변경 이력

[PUB-CK-05] ✅ 개인정보 처리 준수
             - 개인정보 수집 최소화
             - 개인정보 마스킹 (화면 표시)
             - 개인정보 파기 정책

[PUB-CK-06] ✅ 행정 리포팅
             - 현황 통계 (일/주/월/연)
             - 감사 대응 리포트 (기간별, 항목별)
             - 엑셀/PDF 내보내기

[PUB-CK-07] ✅ 공지사항 / 알림 기능
             - 시스템 공지 (팝업, 배너)
             - 개인별 알림 (결재 요청, 마감 임박)

[PUB-CK-08] ✅ 법정 보존 기간 관리
             - 문서/기록 유형별 보존 기간 설정
             - 보존 기간 만료 알림 및 파기 승인 흐름

[PUB-CK-09] ✅ 연계 시스템 식별
             - 행정전산망 (GIMS, SEAMS 등)
             - 전자인사관리시스템 (e-사람)
             - 통합재정정보시스템 (디브레인)
             - 표준지방재정시스템 (e-호조)

[PUB-CK-10] ✅ 장애인 접근성 검증 계획
             - 오픈 전 접근성 전문 검사 일정
             - 보조기술(스크린리더) 호환 테스트
```

### 흔한 누락 요구사항

```
[PUB-MISS-01] 결재 위임 + 자동 만료
  → "과장님이 출장 중이어도 결재가 안 막혀야 해요"
  → 위임 기능만 생각하고 '위임 종료 후 자동 원복'을 빠뜨림

[PUB-MISS-02] 감사 로그 위변조 방지
  → 감사 로그는 만들었는데 DB 관리자가 수정 가능한 구조
  → 별도 테이블 + 해시 체인 또는 WORM 스토리지 연계

[PUB-MISS-03] 개인정보 다운로드 승인 흐름
  → 대량 개인정보 다운로드 시 관리자 승인 없이 가능
  → 다운로드 사유 입력 + 관리자 사후 모니터링

[PUB-MISS-04] 브라우저 호환성 (인터넷 익스플로러 레거시)
  → 공공기관 내부망 IE11 의존도 여전히 높음
  → 지원 브라우저 버전 명시 및 폴리필 계획

[PUB-MISS-05] 문서 보존 기간 자동 관리
  → 보존 기간이 지났는데 파기가 안 된 문서 증가
  → 파기 대상 자동 분류 + 파기 승인 워크플로우

[PUB-MISS-06] 다중 로그인 세션 제한
  → 같은 계정으로 여러 PC에서 동시 로그인 허용
  → 단일 세션 정책 또는 중복 로그인 경고
```

### 도메인 전용 질문

```
[PUB-DQ-01] 이 시스템에서 발생하는 문서/기록은 공공기록물에 해당하나요?
             (→ 공공기록물 관리에 관한 법률 적용 여부)

[PUB-DQ-02] 행정 감사나 국정감사 자료 제출에 이 시스템 데이터가 사용되나요?
             (→ 감사 대응 리포팅, 일괄 조회 기능 필요)

[PUB-DQ-03] 민원인/국민이 직접 접근하는 외부 공개 화면이 있나요?
             (→ 웹 접근성 필수 + 개인정보 수집 동의)

[PUB-DQ-04] 전자서명(공인인증서/PASS 등)이 필요한 기능이 있나요?
             (→ GPKI or 민간 전자서명 서비스 연동)

[PUB-DQ-05] 예산/회계 처리와 연관된 기능이 있나요?
             (→ 디브레인, e-호조 연동 가능성)
```

### 규정/법규 매핑

```
규정명                          | 적용 항목                    | 체크리스트 연결
-------------------------------|-----------------------------|-----------------
전자정부법 제45조               | 전자적 처리, 전자문서 관리    | PUB-CK-01, 08
개인정보보호법 제15~22조        | 수집·이용·제공·파기          | PUB-CK-05, PUB-NFR-03
정보보안관리기준(ISMS-P)        | 보안 통제 114개 항목         | PUB-NFR-04
웹접근성 지침 2.1 (KWCAG)      | 인식, 운용, 이해, 견고성     | PUB-NFR-01, PUB-CK-10
공공기록물 관리에 관한 법률     | 보존 기간, 파기 절차         | PUB-CK-08
행안부 정보시스템 구축·운영 지침 | 아키텍처, 보안, 운영 표준    | PUB-NFR-02, 05
```

## 도메인 2: 금융 (Finance)

### 개요

```
대상: 은행, 증권, 보험, 캐피탈, 핀테크, 저축은행
핵심 특성: 원장 정합성 필수, 감독기관 보고 의무, 무중단 요구 극단적
주요 법규: 전자금융거래법, 전자금융감독규정, 금융소비자보호법,
           자본시장법, 금융회사 IT 보안 감독 규정
```

### 필수 NFR

```
[FIN-NFR-01] 원장 정합성 (Ledger Consistency)
  - 모든 금전 거래는 원자성 보장 (ACID)
  - 차변 = 대변 불일치 시 즉시 경보
  - 이중 입력/중복 처리 방지 (멱등성 키)
  - 실시간 대사(Reconciliation) 또는 일 마감 대사

[FIN-NFR-02] 성능 (Financial Transaction SLA)
  - 온라인 거래 응답 시간: 3초 이내 (99 percentile)
  - 배치 처리: 일 마감 N시간 이내 완료
  - 피크 TPS 처리: 평상시 × 5배 이상 설계

[FIN-NFR-03] 가용성
  - 핵심 뱅킹: 99.999% (연간 5분 다운 허용)
  - 부수 기능: 99.9% 이상
  - 무중단 배포 (Blue-Green or Rolling) 필수

[FIN-NFR-04] 보안
  - 전자금융감독규정 제37조 (망 분리) 준수
  - FDS (이상금융거래탐지시스템) 연동 또는 자체 구현
  - 개인신용정보 암호화 (AES-256 이상)
  - 접근 통제: 2-Factor Authentication 강제

[FIN-NFR-05] 감사 추적
  - 모든 금융 거래 이력 영구 보존
  - 거래 이력 조회 응답: 5초 이내
  - 전자금융 사고 시 거래 추적 가능 보장

[FIN-NFR-06] 동시성 제어
  - 잔액 업데이트 경쟁 조건 방지 (비관적/낙관적 잠금)
  - 분산 환경에서 거래 일관성 (분산 트랜잭션)
  - 배치와 온라인 동시 실행 충돌 방지
```

### 필수 체크리스트

```
[FIN-CK-01] ✅ 일 마감 배치 흐름 정의
             - 마감 트리거 조건 및 순서
             - 마감 실패 시 롤백 및 재처리 절차
             - 마감 결과 보고서 자동 생성

[FIN-CK-02] ✅ 대사 (Reconciliation) 기능
             - 외부 기관(은행, 카드사) 데이터와 내부 원장 대사
             - 대사 불일치 항목 자동 분류 및 처리 흐름
             - 대사 결과 감사 기록

[FIN-CK-03] ✅ FDS 연동 또는 이상 거래 탐지
             - 이상 패턴 정의 (금액 한도 초과, 야간 거래, 해외 IP 등)
             - 이상 거래 탐지 시 자동 차단 or 알림 흐름
             - FDS 미연동 시 자체 임계값 기반 경보

[FIN-CK-04] ✅ 복합 결제 및 취소 흐름
             - 부분 취소, 전체 취소 모두 처리
             - 취소 시 원장 역전개 (Reversal) 로직
             - 카드/계좌/포인트 혼합 결제 시나리오

[FIN-CK-05] ✅ 금융 규제 보고서
             - 금감원/한은 보고 의무 항목 식별
             - 보고서 생성 자동화 및 제출 이력 관리

[FIN-CK-06] ✅ 거래 한도 관리
             - 1회/1일/1월 거래 한도 설정
             - 한도 초과 차단 + 알림
             - 한도 변경 승인 흐름

[FIN-CK-07] ✅ 원장 정합성 검증 배치
             - 비정기 또는 일별 원장 검증 배치
             - 불일치 발견 시 경보 + 자동/수동 보정 흐름
```

### 흔한 누락 요구사항

```
[FIN-MISS-01] 멱등성 키 (Idempotency Key)
  → 네트워크 재시도로 인한 이중 처리 방지
  → API 요청마다 고유 키 강제, 중복 검출 로직

[FIN-MISS-02] 거래 취소 후 포인트/혜택 환원
  → 결제 취소 처리는 했는데 적립 포인트/할인 혜택 환원 누락

[FIN-MISS-03] 배치 중단 재개 지점 (Checkpoint)
  → 5시간 배치가 4시간 30분에 실패 → 처음부터 재시작
  → 중간 체크포인트 저장 및 재개 기능

[FIN-MISS-04] 금융 감독 기관 정기 점검 대응
  → 점검 기간 중 시스템 모드 전환 (읽기 전용 등)
  → 점검 이력 및 결과 문서화

[FIN-MISS-05] 시간대 처리 (Timezone)
  → 해외 거래 시 UTC vs KST 변환 오류
  → 일 마감 기준 시각 명시 (은행 영업일 기준)

[FIN-MISS-06] 실시간 잔액 조회 vs 확정 잔액
  → 미결 거래 포함 표시 잔액과 실제 사용 가능 잔액 구분
```

### 도메인 전용 질문

```
[FIN-DQ-01] 하루 몇 건의 금융 거래가 발생하나요? 피크 시간대는?
             (→ TPS 설계, 동시성 제어 수준 결정)

[FIN-DQ-02] 외부 금융 기관(은행, 카드사, VAN사)과 연동이 필요한가요?
             (→ 연동 방식, 대사 주기, 오류 처리 정의)

[FIN-DQ-03] 일 마감 처리가 있나요? 마감 완료 목표 시간은?
             (→ 배치 설계 SLA)

[FIN-DQ-04] FDS(이상금융거래탐지)가 이미 있나요, 새로 구축하나요?
             (→ 연동 방식 또는 신규 구현 범위)

[FIN-DQ-05] 금융감독원 보고 의무 항목이 있나요?
             (→ 보고서 자동화 범위)
```

### 규정/법규 매핑

```
규정명                          | 적용 항목                    | 체크리스트 연결
-------------------------------|-----------------------------|-----------------
전자금융거래법 제21조           | 안전성 기준, 전자금융 사고   | FIN-NFR-03, 05
전자금융감독규정 제37조         | 망 분리 의무                 | FIN-NFR-04
금융소비자보호법               | 불완전판매 방지, 고지의무    | FIN-CK-05
개인신용정보보호 (신용정보법)   | 신용정보 암호화, 제3자 제공  | FIN-NFR-04
```

## 도메인 3: 의료 (Healthcare)

### 개요

```
대상: 병원, 의원, 요양기관, 의료기기 제조사, 헬스케어 스타트업
핵심 특성: 환자 안전 최우선, PHI 보호, 의료 표준 준수
주요 법규: 의료법, 개인정보보호법(의료정보 특칙), 의료기기법,
           국민건강보험법, 요양급여기준
```

### 필수 NFR

```
[MED-NFR-01] PHI (Protected Health Information) 보호
  - 의료 개인정보: 진단명, 처방, 검사 결과 등
  - 암호화 저장 및 전송 (AES-256, TLS 1.2 이상)
  - 접근 제한: 담당 의료진 + 환자 본인만 원칙
  - 국내: 의료법 제21조 (진료기록 열람 제한)

[MED-NFR-02] 진료 연속성 (Clinical Continuity)
  - 진료 중 시스템 장애 시 수기 처리 가능 여부
  - 오프라인 처방 출력 기능 (네트워크 단절 시)
  - EMR 장애 시 최소 기능 모드 (CDSS, Order Entry)

[MED-NFR-03] 의료 데이터 표준
  - HL7 FHIR R4 준수 (외부 연동 시)
  - DICOM (의료 영상)
  - SNOMED CT, ICD-10 코드 체계
  - 건강보험심사평가원 표준 코드 (EDI 청구)

[MED-NFR-04] 감사 추적 (의료법 기준)
  - 진료기록 접근 이력: 10년 보존
  - 처방/수정/삭제 이력 완전 기록
  - 개인정보 접근 로그 5년 보존

[MED-NFR-05] 가용성
  - 응급실, 중환자실: 99.99% (연간 52분 다운)
  - 외래/입원: 99.9% 이상
  - 새벽 4~6시 점검 창 허용 여부 확인

[MED-NFR-06] 의료기기 연동 안전성
  - 의료기기 데이터 수신 오류 시 알람 (환자 안전)
  - 수치 범위 이탈 자동 경보 (Critical Value Alert)
  - 처방-투약 이중 확인 (BCMA: Barcode Medication Administration)
```

### 필수 체크리스트

```
[MED-CK-01] ✅ 보험 청구 EDI 연동
             - 건강보험심사평가원 EDI 청구 흐름
             - 청구 오류 재처리 및 반려 처리
             - 청구 결과 조회 및 수납 연계

[MED-CK-02] ✅ 환자 식별 정책 (Patient Identification)
             - 동명이인 처리 (생년월일, 성별 필수 확인)
             - 응급 환자 미확인 (Unknown Patient) 처리
             - 환자 병합/분리 이력 관리

[MED-CK-03] ✅ 의약품 안전 (Drug Safety)
             - 복약 금기 체크 (Drug-Drug Interaction)
             - 알레르기 경고
             - 최대 용량 초과 경고
             - 소아/노인 용량 보정 알림

[MED-CK-04] ✅ 진료기록 보존 기간
             - 진료기록부: 10년
             - 처방전: 2년
             - 수술기록: 10년
             - 보존 기간 만료 전 알림 및 연장/파기 승인

[MED-CK-05] ✅ 동의서 관리
             - 수술/시술 동의서 전자 서명
             - 개인정보 수집·이용 동의
             - 동의 철회 기능 및 이력 관리

[MED-CK-06] ✅ 환자 접근권 보장
             - 환자 본인 진료기록 열람 및 사본 요청
             - 대리인 요청 처리 (법정대리인, 위임장)

[MED-CK-07] ✅ Critical Value Alert
             - 검사 결과 위험 수치 자동 탐지
             - 담당 의사/간호사 즉시 알림 (앱 푸시, SMS)
             - 알림 수신 확인 이력 (미수신 시 escalation)
```

### 흔한 누락 요구사항

```
[MED-MISS-01] 응급 환자 미확인 처리
  → 의식 없는 환자의 의무 기록 생성 방법 미정의
  → Unknown Patient ID 체계 + 추후 연결(Link) 기능

[MED-MISS-02] 의약품 대체 처방
  → 처방한 약이 재고 없을 때 대체 약 처방 흐름 미정의
  → 약사/의사 협의 후 대체 처방 기록 관리

[MED-MISS-03] 다기관 데이터 공유 동의
  → 의뢰/전원 시 타 기관 진료 정보 공유 동의 누락
  → 환자 동의 기반 의료 정보 교류 (HIE) 요구사항

[MED-MISS-04] 야간/주말 당직 의사 처방 권한
  → 담당의가 없는 시간 처방 권한 예외 처리 미정의
  → 당직 의사 임시 권한 + 담당의 사후 확인 흐름

[MED-MISS-05] 수술실 무균 환경 스캔
  → 수술실 안에서 바코드/RFID 스캔 요구 미언급
  → 장갑 착용 상태에서의 조작 방법 (음성, 풋 페달)
```

### 도메인 전용 질문

```
[MED-DQ-01] 건강보험 청구가 발생하나요? EDI 청구 연동이 필요한가요?

[MED-DQ-02] 의료 영상(X-Ray, CT, MRI)을 시스템에서 처리해야 하나요?
             (→ DICOM 서버, PACS 연동 여부)

[MED-DQ-03] 응급실 운영이 포함되나요?
             (→ 미확인 환자, 24시간 가용성 요구사항)

[MED-DQ-04] 타 병원/의원과 진료 정보를 공유해야 하나요?
             (→ HL7 FHIR 연동, 의뢰/전원 흐름)

[MED-DQ-05] 임상시험(CRF) 또는 연구 목적 데이터 수집이 있나요?
             (→ IRB 승인, 익명화/가명화 처리)
```

### 규정/법규 매핑

```

-------------------------------|-----------------------------|-----------------

의료법 제22조                   | 진료기록 보존 기간           | MED-CK-04
```

## 도메인 4: 제조 (Manufacturing)

### 개요

```
대상: 생산 현장, 스마트 팩토리, 설비 관리, 품질 관리
핵심 특성: 실시간 센서 데이터, OT/IT 융합 보안, 로트 추적, 오프라인 내성
주요 연동: ERP (SAP, Oracle), MES, SCADA, PLC, IoT 센서
```

### 필수 NFR

```
[MFG-NFR-01] 실시간 데이터 처리
  - 센서 데이터 수집 지연: 1초 이내
  - 이상 감지 경보 발생: 탐지 후 5초 이내
  - 대용량 시계열 데이터 저장 (분당 수천 포인트)

[MFG-NFR-02] 오프라인 내성 (Offline Resilience)
  - 네트워크 단절 시 로컬 데이터 버퍼링 (최소 24시간)
  - 연결 복원 시 자동 동기화 (중복 없이)
  - 현장 작업자 최소 기능 지속 (생산 지시 조회 등)

[MFG-NFR-03] OT 보안 (Operational Technology)
  - IT-OT 망 분리 또는 DMZ 구성
  - SCADA/PLC 접근 로그 기록
  - 이상 패킷 탐지 (IDS for ICS)
  - USB 포트 제한, 화이트리스트 기반 소프트웨어

[MFG-NFR-04] 로트/배치 추적성 (Traceability)
  - 원자재 → 반제품 → 완제품 완전 추적
  - 역추적 (Recall 대응): 특정 로트의 원자재 추적
  - GS1 표준 바코드/RFID 지원

[MFG-NFR-05] 설비 가용성 (OEE)
  - 설비 가동률 실시간 모니터링
  - 예방보전 스케줄 관리
  - 고장 발생 시 평균 수리 시간 (MTTR) 기록
```

### 필수 체크리스트

```
[MFG-CK-01] ✅ ERP 연동 인터페이스 정의
             - 생산 계획 수신 (ERP → MES)
             - 생산 실적 전송 (MES → ERP)
             - 자재 소비 실적 전송
             - 연동 주기 및 오류 처리

[MFG-CK-02] ✅ 로트 추적 흐름
             - 입고 → 검사 → 공정 → 완제품 → 출하 전 단계 추적
             - 불량 발생 시 관련 로트 자동 격리 기능

[MFG-CK-03] ✅ 품질 검사 결과 관리
             - 검사 기준 설정 (UCL, LCL, 합/불 판정)
             - SPC (통계적 공정 관리) 차트 생성
             - 불량 유형별 분류 및 파레토 분석

[MFG-CK-04] ✅ 설비 점검 이력 관리
             - 예방보전 주기 등록 및 알림
             - 점검 결과 입력 및 사진 첨부
             - 보전 이력 조회 및 분석

[MFG-CK-05] ✅ 생산 실적 집계
             - 교대조별 생산 실적
             - 목표 대비 실적 비교 (일/주/월)
             - 손실 시간 유형별 집계 (계획/비계획 정지)

[MFG-CK-06] ✅ 현장 작업 지시
             - 작업 지시서 출력 (현장 프린터)
             - QR/바코드 스캔으로 작업 시작/완료 기록
             - 작업 순서 변경 실시간 반영
```

### 흔한 누락 요구사항

```
[MFG-MISS-01] 교대조 인수인계
  → 조별 인수인계 사항 기록 및 확인 서명 기능 누락
  → 미결 작업, 설비 이상, 특이 사항 이월 흐름

[MFG-MISS-02] 악성 자재 격리 (Quarantine) 시 재고 이동
  → 불량 자재 격리 처리 시 재고 시스템 반영 누락

[MFG-MISS-03] 설비 OEE 분모 기준 합의
  → 설비 가동률 계산 시 분모(계획 시간) 기준 미정의

[MFG-MISS-04] 다중 공장/라인 전환 시 권한
  → 작업자가 다른 라인 데이터에 접근하는 케이스 미정의
  → 공장/라인별 권한 분리 정책

[MFG-MISS-05] IoT 센서 이상 vs 공정 이상 구분
  → 센서 고장과 실제 공정 이상을 동일하게 처리
  → 센서 건강 상태 모니터링 + 이상 원인 구분 알람
```

```
[MFG-DQ-02] 현장 설비에서 직접 데이터를 수집해야 하나요?
```

## 도메인 5: 유통 (Retail / Distribution)

### 개요

```
대상: 이커머스, 오프라인 유통, 물류센터, 프랜차이즈, 편의점
핵심 특성: 실시간 재고 동기화, 복합 결제, 프로모션 복잡성, 반품/교환 흐름
주요 연동: PG사, 카드사, 물류사(CJ대한통운 등), ERP, WMS
```

### 필수 NFR

```
[RET-NFR-01] 결제 시스템 안정성
  - PG 연동 결제 성공률: 99.5% 이상
  - 결제 응답 시간: 5초 이내 (사용자 경험)
  - 결제 실패 시 자동 재시도 및 명확한 실패 원인 안내
  - 이중 결제 방지 (멱등성 키)

[RET-NFR-02] 재고 동기화
  - 온/오프라인 채널 간 재고 실시간 동기화 (5분 이내)
  - 초과 판매 (Overselling) 방지
  - 재고 0 도달 시 자동 품절 처리 및 알림

[RET-NFR-03] 피크 타임 성능
  - 블랙프라이데이, 연말 세일 등 10배 트래픽 처리
  - 자동 스케일 아웃 또는 사전 리소스 확보 계획
  - 피크 시 응답 시간 기준 완화 정책 합의

[RET-NFR-04] 개인정보 (고객 데이터)
  - 결제 정보: PCI-DSS 준수 (카드 정보 직접 저장 금지)
  - 구매 이력 개인정보: 보존 기간 및 파기 정책
  - 마케팅 수신 동의 관리

[RET-NFR-05] 가용성
  - 주문/결제 서비스: 99.9% 이상
  - 단, 새벽 4~6시 점검 창 허용 여부 확인
  - 결제 실패 시 장바구니 보존 (세션 복원)
```

### 필수 체크리스트

```
[RET-CK-01] ✅ PG 연동 및 복합 결제
             - 지원 결제 수단 목록 (신용카드, 간편결제, 계좌이체, 가상계좌)
             - 복합 결제 (포인트 + 카드 등)
             - 부분 결제 (정기 배송, 분할 납부)
             - 해외 결제 (다통화, 환율 처리)

[RET-CK-02] ✅ 반품/교환/환불 흐름
             - 전체 반품 / 부분 반품
             - 교환 후 차액 처리 (추가 결제 or 환불)
             - 배송 상태에 따른 취소 가능 여부
             - 반품 배송비 처리 (판매자 부담 / 구매자 부담)

[RET-CK-03] ✅ 프로모션 엔진
             - 쿠폰 (정률/정액, 사용 조건, 중복 적용 여부)
             - 포인트 적립/사용 정책
             - 회원 등급별 혜택
             - 번들/세트 할인, 조건부 할인 (N개 이상 구매)

[RET-CK-04] ✅ 재고 관리
             - 창고별/지점별 재고 현황
             - 안전 재고 설정 및 알림
             - 재고 조정 이력 (실사 후 조정)
             - 가상 재고 (예약/보류 재고) 처리

[RET-CK-05] ✅ 배송 추적
             - 주문 → 출고 → 배송 → 완료 상태 관리
             - 물류사 API 연동 (CJ, 한진, 롯데택배 등)
             - 배송 지연 알림 자동 발송

[RET-CK-06] ✅ 정산 처리
             - 판매자(입점사) 정산 주기 및 계산 로직
             - 수수료, 반품 공제 처리
             - 세금계산서 자동 발행 연동

[RET-CK-07] ✅ 회원 관리
             - 소셜 로그인 (카카오, 네이버, 애플 등)
             - 비회원 구매 허용 여부
             - 회원 탈퇴 시 개인정보 처리 (즉시 삭제 vs 보존 후 파기)
```

### 흔한 누락 요구사항

```
[RET-MISS-01] 품절 후 재입고 알림 구독
  → 품절 처리는 했는데 재입고 시 대기 고객 알림 기능 누락

[RET-MISS-02] 쿠폰 중복 적용 우선순위 규칙
  → 여러 쿠폰 적용 시 할인 순서/중복 여부 정책 미정의
  → 최대 할인 자동 적용 vs 사용자 선택 정책

[RET-MISS-03] 해외 직구 세관 신고 금액
  → 해외 직구 시 세관 신고 금액 처리 정책 누락
  → 과세 기준 초과 시 안내 문구 및 책임 고지

[RET-MISS-04] 단종/판매 중단 상품의 리뷰/정보
  → 단종 처리 후 해당 상품 URL/리뷰 접근 불가
  → 단종 상품 페이지 처리 정책 (삭제/숨김/대체 안내)

[RET-MISS-05] 입점사 세금계산서 발행 예외
  → 면세 사업자, 간이과세자에 대한 정산 처리 정책 미정의

[RET-MISS-06] 환불 완료 후 포인트 사용분 처리
  → 포인트로 구매한 상품 환불 시 현금 환불 or 포인트 복원?
```

### 도메인 전용 질문

```
[RET-DQ-01] 온라인 채널만인가요, 오프라인(POS) 채널도 있나요?
             (→ 옴니채널 재고 동기화 범위)

[RET-DQ-02] 입점 판매자(셀러)가 있는 마켓플레이스 구조인가요?
             (→ 판매자 정산, 수수료 정책, 에스크로)

[RET-DQ-03] 해외 배송이 있나요?
             (→ 다통화, 관세, 국제 배송사 연동)

[RET-DQ-04] 정기 배송(구독) 서비스가 있나요?
             (→ 자동 결제, 배송 주기 관리, 해지 흐름)

[RET-DQ-05] 명절/블랙프라이데이 같은 대형 프로모션 일정이 있나요?
             (→ 피크 트래픽 설계, 선착순 처리)
```
