# Changelog Viewer - Output Formats

> This file defines the detailed output formats for the `/changelog` skill.

---

## Basic Report (Change History List)

```
===========================================================
  Changelog: <filter description>
===========================================================

  Period: <start-date> ~ <end-date>
  Total: <N entries>

-----------------------------------------------------------
  2026-02-07 (3 entries)
-----------------------------------------------------------
  [Feature] Create discount_service in order
     domain: order
     files:  order/services/discount_service.py
             order/schemas/discount.py
     impact: external dependency added: member-api

  [Fix] Fix issue in payment_service
     domain: payment
     files:  payment/services/payment_service.py
     impact: none

  [Refactor] Refactor order_service
     domain: order
     files:  order/services/order_service.py
     impact: none

-----------------------------------------------------------
  2026-02-06 (1 entry)
-----------------------------------------------------------
  [Test] Update test test_discount.py
     domain: order
     files:  tests/order/test_discount.py
     impact: none

===========================================================
```

## When No Changes Are Found

```
===========================================================
  Changelog: <filter description>
===========================================================

  No changes found.

  Possible reasons:
  - The changelog-recorder.js hook has not run yet
  - No YAML files exist in the .claude/changelog/ directory
  - No changes match the specified filter conditions

  Tip: Modifying source files will cause the hook to automatically record change history.

===========================================================
```

---

## YAML Entry Format

YAML format generated by `changelog-recorder.js`:

```yaml
entries:
  - date: 2026-02-07T14:30:00
    type: feature
    domain: order
    files:
      - order/services/discount_service.py
      - order/schemas/discount.py
    description: "Create discount_service in order"
    impact:
      - "external dependency added: member-api"
```

---

## Statistics Calculation Method

### Bar Chart Representation

```
max value    = the item with the highest count
max bar length = 12 characters (#)
bar per item = round(count / max_value * 12)
minimum 1 character guaranteed (if count > 0)
```
